计数类DP

我们通过几道例题来介绍动态规划中的计数问题和数位统计问题。这两类问题都特别强调 “不重不漏”, 统计对象的可能情况一般比较多, 通常需要精确的划分和整体性的计算。因此, 使用动态规划抽象出问题中的 “子结构” 和推导 的 “阶段”, 将有助于我们准确而高效地进行求解。

306. 杰拉尔德和巨型象棋

给定一个 H×WH \times WH×W 的棋盘，棋盘上只有 NNN 个格子是黑色的，其他格子都是白色的。

在棋盘左上角有一个卒，每一步可以向右或向下移动一格，并且不能移动到黑色格子中。

求这个卒从左上角移动到右下角，一共有多少种路线。

输入格式

第一行包含三个整数 H,W,NH,W,NH,W,N。

接下来 NNN 行，每行包含两个整数 x，yx，yx，y，描述一个黑色格子位于 xxx 行 yyy 列。

数据保证左上角和右下角的格子都是白色的。

输出格式

输出一个整数表示结果对 109+710^9+7109+7 取模后的值。

数据范围

1≤H,W≤1051 \le H,W \le 10^51≤H,W≤105,
1≤N≤20001 \le N \le 20001≤N≤2000

算法分析

棋盘上的白色格子数量巨大, 而黑色格子最多只有 2000 个。我们应该考虑如何依靠黑色格子进行计数。如果这个卒能走进黑色格子, 那么从左上角走到右下角的路线总数就是 CH+W−2H−1C_{H+W-2}^{H-1}CH+W−2H−1​, 我们在 “组合计数” 中提到过这个公式。若我们再求出从左上角到右下角经过了至少一个黑色格子的路线数量, 二者相减就得到了只经过白色格子的路线数。

把所有黑色格子按照行、列坐标递增的顺序排序。特别地, 我们假设左上角是第 0 个黑色格子, 右下角是第 N+1N+1N+1 个黑色格子。设第 iii 个黑色格子在第 xix_{i}xi​ 行第 yiy_{i}yi​ 列。设 F[i]F[i]F[i] 表示从左上角走到排序后的第 iii 个黑色格子, 并且途中不经过其他黑色格子的路线数。

F[i]=Cxi−1+yi−1xi−1−∑j=0i−1F[j]∗Cxi−xj+yi−yjxi−xj, 其中 xi≥xj,yi≥yjF[i]=C_{x_{i}-1+y_{i}-1}^{x_{i}-1}-\sum_{j=0}^{i-1} F[j] * C_{x_{i}-x_{j}+y_{i}-y_{j}}^{x_{i}-x_{j}}, \text { 其中 } x_{i} \geq x_{j}, y_{i} \geq y_{j} F[i]=Cxi​−1+yi​−1xi​−1​−j=0∑i−1​F[j]∗Cxi​−xj​+yi​−yj​xi​−xj​​, 其中 xi​≥xj​,yi​≥yj​

上式的含义是, 枚举 jjj 作为从左上角到 (xi,yi)\left(x_{i}, y_{i}\right)(xi​,yi​) 经过的第一个黑色格子。也就是从左上角到 (xj,yj)\left(x_{j}, y_{j}\right)(xj​,yj​) 不能经过其他黑格子, 路线数为 F[j]F[j]F[j] 。从 (xj,yj)\left(x_{j}, y_{j}\right)(xj​,yj​) 到 (xi,yi)\left(x_{i}, y_{i}\right)(xi​,yi​) 随便走, 路线数就是一个组合数。等式中求和符号的部分求出了从左上角到 (xi,yi)\left(x_{i}, y_{i}\right)(xi​,yi​), 途中经过至少一个黑色格子的路线数, 用总数减掉, 就得到了 F[i]F[i]F[i] 。

我们为什么这样设计状态转移方程? 为什么限制 jjj 作为经过的第一个黑色格子呢? 请读者回顾 “区间 DP” 中的 “金字塔”一题, 在该题中我们提到, 一个状态划分成的若干个子问题之间应该具有互斥性, 才不会造成重复统计。实际上, 我们在求解计数类动态规划问题时, 通常要找到一个 “基准点”, 围绕这个基准点构造一个不可划分的 “整体”, 以避免子问题之间的重叠。既然需要求出从左上角到 (xi,yi)\left(x_{i}, y_{i}\right)(xi​,yi​) 途中经过至少一个黑色格子的路线数, 就枚举第一个经过的黑色格子 jjj, 使左上角到 jjj 构成一个整体, 让这段路程只能经过白色格子, 不能再进行拆分。因为第一个经过的黑色格子不同, 所以不同的 jjj 对应的路线肯定不会重复。而 jjj 又取遍了 iii 之前的所有黑色格子, 所以路线也不会遗漏。结合乘法原理和加法原理, 我们就得到了状态转移方程中求和符号的部分。

以 F[0]=1F[0]=1F[0]=1 为初值, 计算上面的状态转移方程, 最终 F[N+1]F[N+1]F[N+1] 就是本题的答案。 对于组合数, 可以预处理阶乘关于 109+710^{9}+7109+7 的乘法逆元来计算。

算法分析

在计数类动态规划中, 通常要把一个问题划分成若干个子问题, 以便于执行递推。一个连通图不容易进行划分, 而一个不连通的无向图则很容易划分成节点更少的两部分。所以我们可以考虑求出 NNN 个点的无向图总数, 减去 NNN 个点的不连通无向图的数量, 就是 NNN 个点的连通无向图个数。

NNN 个点的无向图总数显然是 2N∗(N−1)/22^{N *(N-1) / 2}2N∗(N−1)/2 。含义是 NNN 个点的无向图至多有 N∗(N−1)/2N * (N-1) / 2N∗(N−1)/2 条边, 每条边可有可无, 所以共有 2N∗(N−1)/22^{N *(N-1) / 2}2N∗(N−1)/2 张不同的无向图。

接下来计算 NNN 个点的不连通无向图的数量。一张不连通的无向图必定由若干个连通块构成。根据上一题提到的 “围绕基准点构造一个整体” 的思想, 我们可以枚举标号为 1 的节点所在的连通块包含的节点个数 kkk, 从 2∼N2 \sim N2∼N 这 N−1N-1N−1 个节点中选出 k−1k-1k−1 个, 与 1 号节点一起构成大小为 kkk 的连通块。显然, 我们有 CN−1k−1C_{N-1}^{k-1}CN−1k−1​ 种选法。剩余 N−kN-kN−k 个节点构成任意无向图, 有 2(N−k)∗(N−k−1)/22^{(N-k) *(N-k-1) / 2}2(N−k)∗(N−k−1)/2 种方法。

综上所述, 设 F[i]F[i]F[i] 表示 iii 个节点的无向连通图个数, 状态转移方程为:

F[i]=2i∗(i−1)/2−∑j=1i−1F[j]∗Ci−1j−1∗2(i−j)∗(i−j−1)/2F[i]=2^{i *(i-1) / 2}-\sum_{j=1}^{i-1} F[j] * C_{i-1}^{j-1} * 2^{(i-j) *(i-j-1) / 2} F[i]=2i∗(i−1)/2−j=1∑i−1​F[j]∗Ci−1j−1​∗2(i−j)∗(i−j−1)/2

初值: F[1]=1F[1]=1F[1]=1 。目标: F[N]F[N]F[N] 。 .

309. 装饰围栏

有 NNN 块长方形的木板，长度分别为 1,2,…,N1,2,…,N1,2,…,N，宽度都是 111。

现在要用这 NNN 块木板组成一个宽度为 NNN 的围栏，满足在围栏中，每块木板两侧的木板要么都比它高，要么都比它低。

也就是说，围栏中的木板是高低交错的。

我们称“两侧比它低的木板”处于高位，“两侧比它高的木板”处于低位。

显然，有很多种构建围栏的方案。

每个方案可以写作一个长度为 NNN 的序列，序列中的各元素是木板的长度。

把这些序列按照字典序排序，如下图所示，就是 N=4N=4N=4 时，所有满足条件的围栏按照木板长度的字典序排序后的结果。

现在给定整数 CCC，求排名为 CCC 的围栏中，各木板的长度从左到右依次是多少。

输入格式

第一行包含整数 KKK，表示一共有 KKK 组数据。

接下来 KKK 行，每行包含一组数据，包括两个整数 NNN 和 CCC。

输出格式

每组数据输出一行结果，结果表示排名为 CCC 的围栏中，各木板的长度从左到右排成的序列。

同行数据用空格隔开。

数据范围

1≤N≤201 \le N \le 201≤N≤20,
0<C<2630 < C < 2^{63}0<C<263

输入样例：

输出样例：

算法分析

类似于倍增优化 DP 中的 “拼凑” 思想, 我们可以用 “试填法” 来确定排名为 CCC 的栅栏中各木板的长度。具体来说, 我们可以从小到大枚举第一块木板的长度, 当第一块木板长度为 hhh 时, 设后面 N−1N-1N−1 块木板构成栅栏的方案数为 TTT, 若 T≥CT \geq CT≥C, 则说明第一块木板的长度就应该是 hhh, 可以继续尝试确定第二块木板的长度。否则, 说明第一块木板应该更长, 此时令 CCC 减去 T,hT, hT,h 增加 1 , 重复上述判断。为了让 “试填法” 更加高效, 我们需要预处理 TTT 的值。

设 F[i,j,k]F[i, j, k]F[i,j,k] 表示用 iii 块长度不同的木板构成栅栏, 其中最左边的木板的长度从小到大排在第 jjj 位, 并且位置状况为 kkk （k=0k=0k=0 表示处于低位, k=1k=1k=1 表示处于高位）的方案总数。状态转移方程为:

F[i,j,0]=∑p=ji−1F[i−1,p,1]F[i,j,1]=∑p=1j−1F[i−1,p,0]\begin{align} F[i, j, 0]=\sum_{p=j}^{i-1} F[i-1, p, 1]\\ F[i, j, 1]=\sum_{p=1}^{j-1} F[i-1, p, 0] \end{align} F[i,j,0]=p=j∑i−1​F[i−1,p,1]F[i,j,1]=p=1∑j−1​F[i−1,p,0]​​

注意, 在动态规划的状态表示中, 我们没有采取像 “用长度为 1∼i1 \sim i1∼i 的木板、最左边的长度为 jjj ” 这种长度绝对确定的表述, 而是把 iii 和 jjj 依据相对大小顺序进行定义。这是因为以下两个子问题其实是等价的:

  1. 用长度为 1∼i1 \sim i1∼i 的木板构成栅栏, 其中最左边的长度为 jjj 。
  2. 用 iii 块长度不同的木板构成栅栏, 其中最左边的长度排名为 jjj 。

实际上我们之前多次使用的 “离散化” 思想, 就与上面这种 “等效代换” 思想的本质是相同的。利用上面两个子问题的等价性, 我们在进行状态转移,数量统计时会非常容易。

在完成动态规划预处理后, 就可以执行 “试填法” 了。

假设已经确定了从左到右前 i−1i-1i−1 块木板的长度, 现在正在考虑第 iii 块木板。记
录上一块木板的长度 lastlastlast 、上一块木板的高低位置情况 kkk （ k=0k=0k=0 表示低位, k=1k=1k=1 表示高位。）

把 kkk 赋值为 k  xor  1k \;xor\; 1kxor1 , 即可得到第 iii 块木板应该处于高位还是低位。

特别地, 第 1 块木板既可能处于高位, 也可能处于低位。所以要注意单独处理, 先确定下来第 1 块木板的高度和位置情况, 作为 lastlastlast 和 kkk 的初始值。

从小到大枚举第 iii 块木板的长度, 设它的真实长度为 lenlenlen , 该长度在剩余木板中从小到大排名为 jjj 。若 kkk 为 0 , 需要满足 len<lastlen < lastlen<last , 否则应满足 len>lastlen > lastlen>last。

若 F[N−i+1,j,k]<CF[N-i+1, j, k]<CF[N−i+1,j,k]<C, 则令 CCC 减掉 F[N−i+1,j,k]F[N-i+1, j, k]F[N−i+1,j,k], 继续尝试更大的 jjj 。

否则, 第 iii 块木板的长度就应该是 lenlenlen , 可以把它输出。然后, 令 iii 增加 1 , 考虑下一块木板。

最后, 我们就得到了所求的栅栏方案。动态规划的时间复杂度为 O(N3)O\left(N^{3}\right)O(N3), 试填过程的时间复杂度为 O(N2)O\left(N^{2}\right)O(N2) 。
